# 기획서 검증 에이전트 (Proposal Validation AI Agent)

> **"발표 전에 가장 아플 질문을 미리 맞아본다."**
> 실제 발표나 중간평가에 들어가기 전, 다중 페르소나 AI 심사위원단(투자자, CTO, 멘토)이 연달아 날카로운 압박 질문과 기술적 허점을 찔러주는 대화형 모의 심사(레드팀) 서비스.

## 프로젝트 배경 및 문제 정의
대부분의 프로젝트(특히 SW마에스트로, 해커톤 등)에서 탈락과 재설계가 발생하는 지점은 '구현'이 아닌 **'기획'**이다. 
* **자기 객관화의 한계:** 작성자는 이미 자기 논리에 설득되어 시장성, 기술 실현 가능성의 빈틈을 발견하기 어렵다.
* **사전 검증 기회 부족:** 본 발표 전에 심사위원급의 퀄리티 높은 압박 질문을 받아볼 기회가 현저히 적다.
* **피드백의 편차:** 리뷰어의 역량에 따라 피드백의 깊이가 달라 치명적인 약점을 놓치기 쉽다.

**해결책:** 정형화된 기획서를 LLM 에이전트가 분석하여, 기획의 약점을 선제적으로 도출하고 방어 논리를 훈련할 수 있는 자동화된 시스템을 구축했다.

---

## 다중 페르소나 심사위원단 (Agent Roles)
단일 챗봇이 아닌, 역할이 완벽히 분리된 **다중 페르소나 에이전트(Multi-Agent)** 아키텍처를 채택했다.

| 페르소나 | 주요 평가 관점 | 대표 질문 예시 |
| :--- | :--- | :--- |
| 🤖 **오케스트레이터** | 파싱 및 작업 분배 | "기획서 분석을 완료했습니다. 모의 심사를 시작합니다." |
| 💼 **깐깐한 투자자** | 시장성, 차별성, 수익/BM | "이거 이미 OOO 서비스가 꽉 잡고 있는데, 왜 지금 당장 필요한가요?" |
| 💻 **냉철한 CTO** | 기술 타당성, 아키텍처, 구현 | "이 기능을 굳이 LLM으로 해야 하나요? 주어진 기간 내 구현 가능한가요?" |
| 🦉 **예리한 멘토** | 논리 일관성, MVP 범위 | "문제 정의와 핵심 기능이 따로 놉니다. MVP가 너무 방대한데 뭘 버릴 건가요?" |

## 핵심 기능 및 워크플로우 (Core Features)

1. **기획서 업로드 및 파싱 (Parser)**
   * TXT/PDF 기반의 기획서를 업로드하면, 문제 정의, 타깃, 핵심 기능, 기술 스택 등의 섹션 단위로 컨텍스트를 구조화한다.
2. **릴레이 압박 면접 (Agentic Workflow)**
   * 오케스트레이터가 섹션 데이터를 각 페르소나에 분배한다.
   * `투자자 → CTO → 멘토` 순서로 사용자의 방어 논리를 꼬리를 물고 파고드는 5~6턴의 핑퐁 압박 질문을 생성한다.
3. **최종 종합 리포트 출력 (Final Report)**
   * 모든 질의응답이 끝나면, 논리적/기술적 허점을 종합하여 **위험도(상/중/하) 및 구체적인 보완 제안(Action Item)**을 한 화면에 리포팅한다.

## 기술 스택 및 아키텍처 (Tech Stack)

* **Language:** Python 3.11
* **AI & Agent Framework:** LangGraph (다중 에이전트 상태 그래프 제어), LangChain
* **LLM:** Upstage Solar-Pro / Anthropic Claude (추상화를 통한 모델 교체 용이성 확보)
* **Frontend:** Streamlit (로컬 데모 및 실시간 SSE 스트리밍 통신 구현)

### System Flow
`[업로드 UI]` ➡️ `[텍스트 전처리/파서]` ➡️ `[오케스트레이터(LangGraph)]` ➡️ `[투자자/CTO/멘토 에이전트 순차 순회]` ➡️ `[종합 리포터]` ➡️ `[리포트 UI]`

> **프롬프트 설계 전략:** 각 에이전트에게 외부 사실에 대한 임의 단정을 금지(환각 통제)하고, 출력은 JSON 스키마로 강제하여 UI 렌더링의 안정성을 확보함.

## 목표 및 향후 로드맵 (Roadmap)

**MVP 성공 지표 (KPI)**
* **정성:** 사용자가 "실제 발표에서 나올 법한 질문이다"라고 동의한 비율 70% 이상 달성.
* **정량:** 기획서 1건당 분석 시간 2분 이내, 유효 압박 질문 5개 이상 연속 도출.
